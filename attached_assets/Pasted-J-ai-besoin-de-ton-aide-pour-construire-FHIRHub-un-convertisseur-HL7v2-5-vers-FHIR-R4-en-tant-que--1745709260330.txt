J'ai besoin de ton aide pour construire FHIRHub, un convertisseur HL7v2.5 vers FHIR R4, en tant que projet autonome sans Docker. FHIRHub est un hub universel qui permet de transformer des messages HL7v2.5 (format français avec INS - Identifiant National de Santé) vers le format FHIR R4.

## Architecture du projet

FHIRHub doit avoir une architecture en deux parties :

1. **Backend (Java/Spring Boot)** 
   - Composant principal gérant la conversion HL7v2.5 vers FHIR R4
   - API REST 
   - Récepteur MLLP
   - Surveillance de dossiers
   - Base de données SQLite pour les logs et les métadonnées

2. **Frontend (React)** 
   - Interface utilisateur moderne et intuitive
   - Dashboard avec statistiques
   - Gestion des applications (ajout, suppression, apikeys)
   - Visualisation des résultats de conversion
   - Upload de fichiers HL7

## Spécifications techniques et exemples de code HAPI FHIR

### Backend
- Java 11
- Spring Boot 2.7.x
- HAPI FHIR 6.6.0 pour la manipulation des ressources FHIR
- HAPI HL7v2 2.3 pour le parsing et la manipulation des messages HL7
- SQLite pour la base de données
- API REST avec authentification par API key
- Support complet du format FR HL7v2.5 incluant l'INS

### Configuration Maven pour HAPI FHIR

```xml
<dependencies>
    <!-- Base HAPI FHIR -->
    <dependency>
        <groupId>ca.uhn.hapi.fhir</groupId>
        <artifactId>hapi-fhir-base</artifactId>
        <version>6.6.0</version>
    </dependency>
    
    <!-- FHIR Structures R4 -->
    <dependency>
        <groupId>ca.uhn.hapi.fhir</groupId>
        <artifactId>hapi-fhir-structures-r4</artifactId>
        <version>6.6.0</version>
    </dependency>
    
    <!-- HAPI HL7v2 Parser -->
    <dependency>
        <groupId>ca.uhn.hapi</groupId>
        <artifactId>hapi-base</artifactId>
        <version>2.3</version>
    </dependency>
    <dependency>
        <groupId>ca.uhn.hapi</groupId>
        <artifactId>hapi-structures-v25</artifactId>
        <version>2.3</version>
    </dependency>
    
    <!-- Validation ressources FHIR -->
    <dependency>
        <groupId>ca.uhn.hapi.fhir</groupId>
        <artifactId>hapi-fhir-validation-resources-r4</artifactId>
        <version>6.6.0</version>
    </dependency>
    
    <!-- FHIR Validator -->
    <dependency>
        <groupId>ca.uhn.hapi.fhir</groupId>
        <artifactId>hapi-fhir-validation</artifactId>
        <version>6.6.0</version>
    </dependency>
</dependencies>
