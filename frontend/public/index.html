API REST

L'API FHIRHub fournit des endpoints pour la conversion HL7 vers FHIR et la gestion des applications.

Toutes les requêtes nécessitent une authentification via l'en-tête x-api-key. Pour les tests, utilisez dev-key.

Endpoints de base

GET /api/status
Vérifier l'état du serveur

Exemple de requête

curl -X GET http://localhost:5000/api/status

Exemple de réponse

{ "status": "ok", "version": "1.0.0", "timestamp": "2025-04-27T23:36:15.577Z" }

Conversion de messages HL7

POST /api/convert
Convertir un message HL7 en FHIR

Exemple de requête

curl -X POST \ -H "Content-Type: application/json" \ -H "x-api-key: dev-key" \ -d '{"content": "MSH|^~\\&|MESA_ADT|XYZ_HOSPITAL|iFW|ABC_LABORATORY|200911091619||ADT^A01|20991109161900001|P|2.5"}' \ http://localhost:5000/api/convert

Exemple de réponse

{ "status": "ok", "data": { "conversion_id": "9eb0f976-5d9e-4c02-9cc7-4e0151a8a65a", "fhir": { "resourceType": "Bundle", "type": "transaction", "entry": [...] } } }
POST /api/upload
Convertir un fichier HL7 téléchargé

Exemple de requête

curl -X POST \ -H "x-api-key: dev-key" \ -F "file=@message.hl7" \ http://localhost:5000/api/upload

Exemple de réponse

{ "status": "ok", "data": { "conversion_id": "e508a3ee-8c65-487d-992f-92d6ac22b984", "fhir": { "resourceType": "Bundle", "type": "transaction", "entry": [...] } } }

Historique et statistiques

GET /api/conversions
Obtenir l'historique des conversions (avec pagination)

Exemple de requête

curl -X GET \ -H "x-api-key: dev-key" \ "http://localhost:5000/api/conversions?limit=10&offset=0"

Exemple de réponse

{ "status": "ok", "data": [ { "id": 5, "conversion_id": "312475f4-3df8-4b12-976c-24ab0eb22c34", "source_type": "API", "status": "success", "created_at": "2025-04-27 20:22:32" } ] }
GET /api/stats
Obtenir les statistiques globales de conversion

Exemple de requête

curl -X GET \ -H "x-api-key: dev-key" \ http://localhost:5000/api/stats

Exemple de réponse

Documentation

Présentation de FHIRHub

FHIRHub est une solution de conversion des messages HL7v2.5 vers le format FHIR R4 (version 4.0.1), conforme aux exigences de l'Agence du Numérique en Santé (ANS).

Cette plateforme est spécialement conçue pour le contexte français de la santé, avec un support intégré des terminologies de santé françaises.

L'application offre une interface utilisateur intuitive tout en proposant une API REST complète pour les intégrations système.

Architecture technique

FHIRHub adopte une architecture légère et portable :

  • Backend : Node.js avec Express pour l'API REST
  • Stockage : Base de données SQLite intégrée pour une portabilité maximale
  • Frontend : Interface utilisateur moderne aux couleurs "e-Santé"
  • Sécurité : Authentification par clé API et système de rôles utilisateurs (admin/lecture seule)
  • Déploiement : Installation directe dans un dossier sans dépendances lourdes
  • Portabilité : Fonctionnement entièrement hors ligne possible

Fonctionnalités principales

Conversion HL7 vers FHIR

  • Conversion complète des messages HL7v2.5 vers FHIR R4
  • Support du format de transaction Bundle FHIR
  • Validation et nettoyage automatique des ressources générées
  • Adaptation aux terminologies françaises ANS
  • Interface de conversion en temps réel avec logs d'exécution
  • Téléchargement direct des fichiers FHIR générés
  • Support des segments HL7 standards et personnalisés

Services de terminologie française

  • Mode hors ligne (par défaut) : Terminologies préchargées pour fonctionnement sans internet
  • Mode en ligne (optionnel) : Connexion directe au Serveur Multi-Terminologies de l'ANS
  • Support complet de l'Identifiant National de Santé (INS-NIR et INS-C)
  • Adaptation automatique des identifiants professionnels (RPPS, ADELI)
  • Support des principaux référentiels français :
    • TRE-R316-AutreCategorieEtablissement
    • TRE-R51-DESCGroupe2Diplome
    • TRE-G02-TypeProduit
    • TRE-R217-ProtectionJuridique
    • TRE-R239-PublicPrisEnCharge
    • TRE-A01-CadreExercice
  • Support des OIDs français standardisés avec validation
  • Système de cache intelligent pour optimiser les performances

Système multi-applications

  • Création et gestion d'applications clientes distinctes
  • Génération de clés API multiples par application (production/qualification)
  • Séparation complète des logs et statistiques par application
  • Configuration personnalisable par application cliente
  • Gestion des rétentions de logs (maximum 1 mois)
  • Sauvegarde facilitée des données par application

Module de nettoyage FHIR

FHIRHub intègre un puissant module de nettoyage des ressources FHIR qui optimise les données générées :

  • Suppression des champs vides : Élimine automatiquement les propriétés non significatives
  • Optimisation des ressources : Restructure les données selon les bonnes pratiques FHIR
  • Correction des références : Résout les problèmes de références relatives
  • Traitement spécifique : Adapte chaque type de ressource de façon optimale
  • Validation de conformité : Vérifie la compatibilité avec les serveurs FHIR français

API REST complète

FHIRHub expose une API REST complète pour intégration système :

API de conversion

  • GET /api/status - Vérifier l'état du serveur
  • GET /api/conversions - Récupérer l'historique des conversions
  • GET /api/conversions/:id - Obtenir une conversion spécifique
  • GET /api/stats - Obtenir les statistiques de conversion
  • POST /api/convert - Convertir du contenu HL7 en FHIR
  • POST /api/upload - Télécharger et convertir un fichier HL7

API de terminologie

  • GET /api/terminology/codesystem/:id - Accéder à un système de code français
  • GET /api/terminology/search - Rechercher dans les terminologies
  • GET /api/terminology/validate - Valider un code dans un système
  • POST /api/terminology/validate-bundle - Valider tous les codes dans un bundle FHIR
  • GET /api/terminology/oid/:oid - Obtenir un système par son OID

API de gestion

  • GET /api/applications - Lister toutes les applications
  • POST /api/applications - Créer une nouvelle application
  • GET /api/applications/:id - Obtenir les détails d'une application
  • PUT /api/applications/:id - Mettre à jour une application
  • DELETE /api/applications/:id - Supprimer une application
  • POST /api/applications/:id/keys - Générer une nouvelle clé API
  • DELETE /api/applications/:id/keys/:keyId - Révoquer une clé API

Guide d'utilisation

Conversion de messages HL7

Deux méthodes sont disponibles pour convertir vos messages HL7 :

  1. Saisie directe : Collez votre message HL7 dans la zone de texte de l'onglet "Convertir" et cliquez sur "Convertir".
  2. Téléchargement de fichier : Utilisez la zone de glisser-déposer pour télécharger un fichier HL7 (.hl7 ou .txt).

La conversion génère trois sections :

  • Résultat FHIR : Le bundle FHIR généré au format JSON
  • Logs de conversion : Détails techniques du processus de conversion
  • Visualisation : Aperçu structuré des ressources générées

Historique des conversions

L'onglet "Historique" vous permet de consulter toutes les conversions effectuées, avec possibilité de filtrer par application, date et statut. Pour chaque conversion, vous pouvez :

  • Visualiser le message HL7 source
  • Consulter le résultat FHIR généré
  • Examiner les logs de conversion détaillés
  • Télécharger les fichiers source et résultat

Gestion des applications

L'onglet "Applications" permet aux administrateurs de créer et gérer des applications clientes. Pour chaque application, vous pouvez configurer :

  • Nom et description de l'application
  • Paramètres de conversion spécifiques
  • Génération et révocation de clés API
  • Limites d'utilisation et quotas
  • Options de rétention des logs

Configuration du service de terminologie

Mode hors ligne (par défaut)

Par défaut, FHIRHub utilise un service de terminologie hors ligne avec des données préchargées :

  • Aucune connexion internet requise
  • Performance optimale sans délai réseau
  • Fonctionnement autonome garanti
  • Idéal pour les environnements sécurisés

Mode en ligne (optionnel)

Pour utiliser le Serveur Multi-Terminologies (SMT) de l'ANS en ligne :

  1. Modifiez le fichier french_terminology_adapter.js pour utiliser le service en ligne
  2. Obtenez vos identifiants auprès de l'ANS
  3. Configurez l'authentification dans le fichier app.js

Le mode en ligne offre l'accès aux terminologies les plus récentes et à l'ensemble complet des référentiels ANS.

Pour des instructions détaillées, consultez la documentation complète des services de terminologie.

Sécurité et authentification

Système d'authentification

FHIRHub implémente un système d'authentification à deux niveaux :

  • Administrateur : Accès complet à toutes les fonctionnalités (identifiants par défaut : admin/adminfhirhub)
  • Utilisateur : Accès en lecture seule aux conversions et données

Sécurité API

L'API REST est sécurisée par un système de clés API :

  • Chaque application dispose de ses propres clés API
  • Possibilité de créer des clés distinctes pour production et qualification
  • Révocation immédiate des clés compromises
  • Traçabilité complète de l'utilisation des clés
