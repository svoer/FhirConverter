Conversion HL7 v2.5 vers FHIR R4

FHIRHub est une solution puissante pour convertir des messages HL7 v2.5 en ressources FHIR R4, optimisée pour les terminologies françaises et conforme aux spécifications de l'ANS.

Conversion Terminologie Système Authentification

APIs de Conversion

POST /api/convert
Convertit un message HL7 v2.5 en ressources FHIR R4 (Bundle Transaction) selon les spécifications de l'ANS.
Paramètres de la requête
hl7Message string requis
Message HL7 v2.5 à convertir

Authentification

        
Toutes les requêtes API doivent être authentifiées en utilisant l'un des deux méthodes suivantes :

        
        
Authentification par clé API

        
Pour les API de conversion, utilisez une clé API en incluant l'en-tête X-API-KEY dans vos requêtes.

        
curl -X POST http://localhost:5000/api/convert \
  -H "Content-Type: application/json" \
  -H "X-API-KEY: votre-clé-api" \
  -d '{"hl7Message": "MSH|..."}'

        
        
Authentification par JWT

        
Pour les API d'administration, utilisez un JWT (JSON Web Token) en incluant l'en-tête Authorization dans vos requêtes.

        
curl -X GET http://localhost:5000/api/stats \
  -H "Authorization: Bearer votre-jwt-token"
